ОПИСАНИЕ ПРОЕКТА
═══════════════════════════════════════════════════════════════════════════

Secure Docs 2FA - это веб-приложение для безопасного хранения и управления
конфиденциальными документами с расширенной системой защиты, включающей
двухфакторную аутентификацию (2FA). Приложение построено на Django и 
предоставляет высокий уровень безопасности для хранения важных файлов.


ОСНОВНЫЕ ХАРАКТЕРИСТИКИ
═══════════════════════════════════════════════════════════════════════════

1. АУТЕНТИФИКАЦИЯ И АВТОРИЗАЦИЯ
   • Регистрация и вход пользователей
   • Разные роли: обычный пользователь и администратор
   • Управление учетными записями с профилями
   • Верификация электронной почты

2. ДВУХФАКТОРНАЯ АУТЕНТИФИКАЦИЯ (2FA)
   • TOTP (Time-based One-Time Password) через Google Authenticator, Authy и т.д.
   • Email OTP - одноразовые коды, отправляемые по электронной почте
   • Резервные коды восстановления доступа
   • Защита от атак перебора (lockout система)

3. УПРАВЛЕНИЕ ДОКУМЕНТАМИ
   • Загрузка и хранение файлов
   • Разметка документов как конфиденциальных
   • История загруженных файлов с датами
   • Описания и заголовки для каждого документа

4. БЕЗОПАСНОСТЬ
   • Логирование всех операций безопасности
   • Отслеживание IP-адресов и User-Agent
   • Мониторинг попыток входа и 2FA
   • Антивирусная защита при неудачных попытках
   • Изоляция документов по владельцам

5. УПРАВЛЕНИЕ ПРОФИЛЕМ
   • Изменение пароля с проверкой текущего пароля
   • Смена электронной почты с верификацией
   • Загрузка аватара профиля
   • Просмотр логов безопасности


ТЕХНОЛОГИЧЕСКИЙ СТЕК
═══════════════════════════════════════════════════════════════════════════

Backend:
  • Django 6.0.3 - веб-фреймворк
  • Python 3.12 - язык программирования
  • PostgreSQL 16 - база данных
  • psycopg 3.3.3 - драйвер PostgreSQL

Безопасность:
  • PyOTP 2.9.0 - реализация TOTP/HOTP
  • QRCode 8.2 - генерация QR-кодов для 2FA
  • Django SecurityMiddleware - защита от типичных уязвимостей

Инструменты:
  • Pillow 12.1.1 - обработка изображений (аватары)
  • Python-dotenv 1.2.2 - управление переменными окружения
  • Colorama 0.4.6 - цветной вывод в консоль

МОДЕЛИ ДАННЫХ
═══════════════════════════════════════════════════════════════════════════

1. UserProfile
   - Связь один-к-одному с Django User
   - Роль (пользователь/администратор)
   - Статус верификации email
   - Аватар профиля

2. Document
   - Принадлежит одному пользователю
   - Заголовок и описание
   - Флаг конфиденциальности
   - Дата загрузки

3. MFASettings
   - Связь один-к-одному с User
   - Статусы TOTP и Email 2FA
   - TOTP секрет
   - Резервные коды
   - Отслеживание неудачных попыток

4. EmailOTPCode
   - Принадлежит одному пользователю
   - Хеш кода
   - Время создания и истечения
   - Статус использования

5. SecurityLog
   - Логирование всех событий безопасности
   - IP-адрес и User-Agent
   - Тип события (вход, 2FA, блокировка и т.д.)
   - Детали события


УСТАНОВКА И ЗАПУСК
═══════════════════════════════════════════════════════════════════════════

1. УСТАНОВКА С ИСПОЛЬЗОВАНИЕМ DOCKER (рекомендуется)

   a) Подготовка:
      - Скопируйте .env.example в .env
      - Заполните переменные окружения:
        * SECRET_KEY - секретный ключ Django
        * DB_NAME, DB_USER, DB_PASSWORD - данные базы данных
        * EMAIL_HOST_USER, EMAIL_HOST_PASSWORD - учетные данные Gmail
      
   b) Запуск:
      docker-compose up -d
      
   c) Приложение будет доступно на http://localhost:8000

2. УСТАНОВКА ЛОКАЛЬНО

   a) Требования: Python 3.12, PostgreSQL
   
   b) Установка зависимостей:
      pip install -r requirements.txt
      
   c) Создание переменных окружения:
      cp .env.example .env
      # Отредактируйте .env с вашими параметрами
      
   d) Миграция базы данных:
      python manage.py migrate
      
   e) Создание суперпользователя:
      python manage.py createsuperuser
      
   f) Запуск сервера разработки:
      python manage.py runserver


ПЕРЕМЕННЫЕ ОКРУЖЕНИЯ
═══════════════════════════════════════════════════════════════════════════

Необходимые переменные в .env:

   SECRET_KEY              - Секретный ключ для шифрования Django сессий
   DEBUG                   - Режим отладки (True/False)
   
   DB_NAME                 - Имя базы данных PostgreSQL
   DB_USER                 - Пользователь базы данных
   DB_PASSWORD             - Пароль базы данных
   DB_HOST                 - Хост базы данных (127.0.0.1 для локального, db для Docker)
   DB_PORT                 - Порт базы данных (5432)
   
   EMAIL_BACKEND           - Backend отправки email
   EMAIL_HOST              - SMTP сервер (например, smtp.gmail.com)
   EMAIL_PORT              - SMTP порт (587)
   EMAIL_HOST_USER         - Email адрес отправителя
   EMAIL_HOST_PASSWORD     - Пароль приложения email
   EMAIL_USE_TLS           - Использование TLS (True)
   DEFAULT_FROM_EMAIL      - Email для From поля


ОСНОВНЫЕ URL МАРШРУТЫ
═══════════════════════════════════════════════════════════════════════════

Аутентификация:
   /accounts/register/          - Регистрация нового пользователя

Двухфакторная аутентификация:
   /security/2fa/setup/         - Настройка 2FA
   /security/2fa/verify/        - Подтверждение 2FA
   /security/2fa/disable/       - Отключение 2FA
   /security/backup-codes/      - Получение резервных кодов

Документы:
   /documents/                  - Список документов пользователя
   /documents/upload/           - Загрузка нового документа
   /documents/<id>/delete/      - Удаление документа

Администрирование:
   /admin/                      - Django admin панель


СИСТЕМА ЛОГИРОВАНИЯ БЕЗОПАСНОСТИ
═══════════════════════════════════════════════════════════════════════════

Все события безопасности логируются:

   login_success      - Успешный вход в систему
   login_failed       - Неудачная попытка входа
   2fa_success        - Успешная верификация 2FA
   2fa_failed         - Неудачная верификация 2FA
   2fa_enabled        - Включена 2FA
   2fa_disabled       - Отключена 2FA
   recovery_code_used - Использован резервный код восстановления
   lockout            - Временная блокировка аккаунта

Каждый лог содержит:
   - IP-адрес пользователя
   - User-Agent браузера
   - Дополнительные детали события

═══════════════════════════════════════════════════════════════════════════

1. Ограничение попыток входа:
   - Максимум 5 попыток входа

   - Логирование всех попыток

2. Защита от перебора 2FA:
   - Отслеживание неудачных попыток верификации

   - Временная разблокировка

3. CSRF защита:
   - Django CSRF middleware
   - Токены для всех форм

4. XSS защита:
   - Django автоматическое экранирование HTML
   - ContentSecurityPolicy заголовки

5. SQL-инъекции:
   - Django ORM предотвращает SQL инъекции
   - Параметризованные запросы


WORKFLOW ИСПОЛЬЗОВАНИЯ
═══════════════════════════════════════════════════════════════════════════

Типичный сценарий:

1. Новый пользователь регистрируется на сайте
2. Верифицирует свою электронную почту
3. Логинится в систему
4. Настраивает 2FA (выбирает TOTP или Email OTP)
5. Подтверждает 2FA настройку
6. Получает резервные коды восстановления
7. Загружает конфиденциальные документы
8. При следующем входе вводит 2FA код
9. Может просматривать логи безопасности
10. Может изменять пароль и email

РАЗВЕРТЫВАНИЕ НА ПРОДАКШЕНЕ
═══════════════════════════════════════════════════════════════════════════

1. Подготовка сервера:
   - Установить Python 3.12
   - Установить PostgreSQL 16
   - Установить Nginx или Apache

2. Переменные окружения:
   - Установить DEBUG = False
   - Использовать надежный SECRET_KEY
   - Настроить ALLOWED_HOSTS
   - Использовать SSL сертификаты

3. Запуск приложения:
   - Использовать Gunicorn/uWSGI вместо runserver
   - Настроить Nginx как reverse proxy
   - Настроить Supervisor/systemd для управления процессом

4. Резервные копии:
   - Регулярно создавать резервные копии БД
   - Хранить на отдельном хранилище
   - Проверять возможность восстановления
